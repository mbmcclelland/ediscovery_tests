dr-tools v0.15.1 — demo tour

Built on 2026-05-14. Branch v0.06, commit dd88958. RPM: dr-tools-0.15.1-1.el9.x86_64.rpm.

Seven screenshots taken by the Textual Pilot harness against the live DR install (192.168.58.128). Each SVG is a faithful render of what your terminal sees — open them directly to inspect the pixel-perfect output.

1. Login

DR sysadmin or organization admin. Cleaner-than-default Textual login dialog; focused field highlighted in cyan.

  • Two roles: DRSysAdmin (system-wide) or admin@training (org-scoped).
  • Password default in lab: password.
  • Enter in either field submits.

2. Landing Dashboard (DRSysAdmin)

Five live panels: license + nodes (REST, 30s refresh); system metrics with sparklines (psutil, 2s); streaming AHS log with INFO/WARN/ERROR filters (1s file poll); top 5 procs (3s).

Traffic-control station. Marcus the beta user specifically wanted this view as his all-day default. CPU / Mem / Net / Disk-IOPS sparklines show peak + average over a rolling 60-sample window.

3. System Settings — Realm Settings sub-tree

Hierarchical tree with two write-paths: Storage depots, Virus Detection, System Users / Groups, and the v0.08+ Realm Settings (mail / splash / password policy / inactivity).

  • F7 New, F4 Edit, F8 Delete on rows that support it.
  • Realm Settings entries open dedicated edit modals (v0.12).
  • Auto-refresh ticks every 5s on the visible leaf only.

4. Job Scheduler — Saved Templates

Three demo templates seeded. Note the * weekly-archive-longterm row — the bold asterisk prefix is the v0.15 accessibility cue for "longterm"-named jobs, visible to colour-blind users regardless of the yellow tint.

Beta-tester win. Marcus (deuteranopia) specifically called this out: "Asterisk is the deciding cue. Thank you for not relying on yellow alone."

Columns: Job Name · Org · Path · Retention · Schedule · Description. Schedule shows preset names (3x-day, weekly) or on-demand in dim text when no recurrence.

5. Run History (v0.15.1 status glyphs)

Each status cell carries a leading UTF-8 glyph in addition to its colour: ▶ RUNNING, ✓ SUCCESS, ✗ FAILURE, ⊘ DELETED/CANCELLED, ‖ PAUSED. v0.15.1 TICKET-2 fix.

v0.15.1 accessibility fix. Pre-glyph version relied on red-vs-green colour alone to distinguish FAILURE from SUCCESS — invisible at a glance for deuteranopia. Now the glyph makes the cue table-scannable even without colour.

6. F3 — realm-wide Jobs Monitor

Single realmManager/listRealmTasks call (v0.11 — replaces the per-project fan-out). Filter buttons All/Running/Completed/Deleted, operation-type Select populated from listOperationTypes, action row for Pause/Resume/Cancel/Priority/Log. L opens the AE live-log viewer.

Press F3 anywhere — the modal pops over whatever tab you're on. Cancel is confirm-gated. Priority opens a 3-button modal (High/Normal/Low, hotkeys h/n/l).

7. New Job wizard

v0.15 layout: form fields left (Name/Description/Organization/Connector/Retention/Schedule), Path Input right. The file-tree browser was dropped in v0.15 in favour of a path Input — matches the proven locustfile_indexing.py pattern and sidesteps DR 5.5.3.2's permission gates on exploreConnector.

Four explicit actions (per original spec): Cancel, Schedule, Run Now, Close. Eight schedule presets including the "3× daily (03/11/19)" preset Marcus needed for the "every day, 3 times a day" use case. The text input below the connector pre-fills with the connector's root path so the user just appends the subfolder.

Quick CLI tour

Beyond the TUI, dr-tools v0.15.1 ships three CLIs. Run any of these from a shell on the lab host:

$ dr-tui                              # the TUI we just toured

$ dr-load preflight                   # env health-check
$ dr-load indexing --users 5 --duration 300s  # Locust load test

$ dr-job-run nightly-payroll          # same code path as TUI's Run Now
$ dr-job-delete nightly-payroll 20260514T030000  # manual retention expire

$ systemctl --user list-timers --all | grep dr-tools

What's new in v0.15.1 vs v0.14.x

  • v0.15.0 — Dropped the file-tree browser (DR 5.5.3.2 doesn't grant exploreConnector permission to any role by default). Added recurring schedules via systemd user timers with 8 presets.
  • v0.15.1 — Beta-tester fixes: ▶ ✓ ✗ ⊘ ‖ glyph prefixes on every status cell (colour-blind accessibility); actionable empty-project hint that names the missing permission.
